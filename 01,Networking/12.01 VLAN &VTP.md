# VLAN (Virtual Local Area Network)

---

## 📌 1. What is a VLAN?

A **VLAN (Virtual Local Area Network)** is a **logical grouping** of devices on a switch,
even if they are not on the same physical switch.

> 🔑 **Simple Definition:**
> VLAN divides **one physical switch** into **multiple virtual switches**.

### Why Use VLAN?

| Benefit            | Description                                      |
|--------------------|--------------------------------------------------|
| **Security**       | Departments can't access each other's traffic     |
| **Performance**    | Reduces broadcast traffic                         |
| **Management**     | Easy to manage network per department             |
| **Cost Saving**    | No need to buy separate switches per department   |

## 📌 4. Without VLAN vs With VLAN

```
  ❌ WITHOUT VLAN                          ✅ WITH VLAN
  ─────────────────                        ─────────────────

  All PCs in ONE                           PCs grouped LOGICALLY
  broadcast domain                         into separate domains

  ┌─────────────────┐                      ┌─────────────────┐
  │  Everyone can   │                      │ SALES ←→ SALES  │
  │  talk to        │                      │ only             │
  │  everyone       │                      ├─────────────────┤
  │                 │                      │ MARKET ←→ MARKET│
  │  📢 Broadcast  │                      │ only             │
  │  goes to ALL   │                      ├─────────────────┤
  │  devices       │                      │ HR ←→ HR only   │
  └─────────────────┘                      └─────────────────┘

  🔴 Not Secure                            🟢 Secure
  🔴 Slow (more broadcast)                 🟢 Fast (less broadcast)
```

## 📌 5. Types of VLAN Ports

### 🔷 Access Port
- Connects to **end devices** (PC, Printer, Server)
- Carries traffic of **only ONE VLAN**
- Switch **adds/removes** VLAN tag automatically

### 🔷 Trunk Port
- Connects **switch to switch** (or switch to router)
- Carries traffic of **MULTIPLE VLANs**
- Uses **802.1Q tagging** to identify VLAN traffic

```
   ACCESS PORT                           TRUNK PORT
   ────────────                          ──────────

   ┌────┐    ┌──────┐                  ┌──────┐    ┌──────┐
   │ PC │────│SWITCH│                  │ SW1  │════│ SW2  │
   └────┘    └──────┘                  └──────┘    └──────┘
              Fa0/1                      Fa0/24      Fa0/24
           (1 VLAN only)              (ALL VLANs pass through)
```

## 📌 9. Trunk Port Configuration

### When Do We Need Trunk?

When we have **2 or more switches** and VLANs must work across them.

```
  SWITCH 1 (SW1)                              SWITCH 2 (SW2)
  ┌──────────────────────┐                    ┌──────────────────────┐
  │                      │   TRUNK LINK       │                      │
  │ Fa0/1  → VLAN 10    │   (Fa0/24)         │ Fa0/1  → VLAN 10    │
  │ Fa0/3  → VLAN 20    │══════════════════   │ Fa0/3  → VLAN 20    │
  │ Fa0/5  → VLAN 30    │   carries ALL       │ Fa0/5  → VLAN 30    │
  │                      │   VLANs (10,20,30) │                      │
  └──────────────────────┘                    └──────────────────────┘

  PC1(Sales)  PC3(Market)  PC5(HR)           PC2(Sales) PC4(Market) PC6(HR)
```

### 🔧 Configure Trunk on SW1

```
SW1(config)# interface fa0/24
SW1(config-if)# switchport mode trunk
SW1(config-if)# switchport trunk allowed vlan 10,20,30
SW1(config-if)# exit
```

### 🔧 Configure Trunk on SW2

```
SW2(config)# interface fa0/24
SW2(config-if)# switchport mode trunk
SW2(config-if)# switchport trunk allowed vlan 10,20,30
SW2(config-if)# exit
```

> 💡 If you want **ALL VLANs** to pass through trunk:
> ```
> SW1(config-if)# switchport trunk allowed vlan all
> ```

### ✅ Verify Trunk

```
SW1# show interfaces trunk
```

**Output:**

```
Port      Mode      Encapsulation  Status     Native vlan
Fa0/24    on        802.1q         trunking   1

Port      Vlans allowed on trunk
Fa0/24    10,20,30

Port      Vlans allowed and active in management domain
Fa0/24    10,20,30
```

## 📌 10. VTP (VLAN Trunking Protocol)

### What is VTP?

VTP **automatically shares VLAN information** between switches
so you don't have to create VLANs on every switch manually.

```
  ┌──────────┐          ┌──────────┐          ┌──────────┐
  │   SW1    │══TRUNK══ │   SW2    │══TRUNK══ │   SW3    │
  │ (SERVER) │          │ (CLIENT) │          │ (CLIENT) │
  └──────────┘          └──────────┘          └──────────┘

  Create VLANs          Automatically          Automatically
  HERE only     ──────► receives VLANs ──────► receives VLANs
```

### VTP Modes

| Mode          | Can Create VLAN? | Learns VLANs? | Forwards VTP? |
|---------------|------------------|----------------|----------------|
| **Server**    | ✅ Yes           | ✅ Yes         | ✅ Yes         |
| **Client**    | ❌ No            | ✅ Yes         | ✅ Yes         |
| **Transparent**| ✅ Yes (local)  | ❌ No          | ✅ Yes         |

### 🔧 VTP Configuration

#### On SW1 (VTP Server)

```
SW1(config)# vtp mode server
SW1(config)# vtp domain MYCOMPANY
SW1(config)# vtp password cisco123
SW1(config)# vtp version 2
```

#### On SW2 (VTP Client)

```
SW2(config)# vtp mode client
SW2(config)# vtp domain MYCOMPANY
SW2(config)# vtp password cisco123
SW2(config)# vtp version 2
```

#### On SW3 (VTP Client)

```
SW3(config)# vtp mode client
SW3(config)# vtp domain MYCOMPANY
SW3(config)# vtp password cisco123
SW3(config)# vtp version 2
```

> ⚠️ **Important:** VTP works **only over trunk links**. Make sure trunk is configured!

### ✅ Verify VTP

```
SW1# show vtp status
```

**Output:**

```
VTP Version                     : 2
Configuration Revision          : 3
Maximum VLANs supported locally : 255
Number of existing VLANs        : 7
VTP Operating Mode              : Server
VTP Domain Name                 : MYCOMPANY
VTP Pruning Mode                : Disabled
```

## 📌 13. Quick Command Reference

| Task                      | Command                                          |
|---------------------------|--------------------------------------------------|
| Create VLAN               | `vlan 10` → `name SALES`                        |
| Show VLANs                | `show vlan brief`                                |

| Show Trunk                | `show interfaces trunk`                          |
| Set VTP Mode              | `vtp mode server/client/transparent`             |
| Set VTP Domain            | `vtp domain MYCOMPANY`                           |
| Set VTP Password          | `vtp password cisco123`                          |
| Show VTP Status           | `show vtp status`                                |
| Delete All VLANs          | `delete flash:vlan.dat` → `reload`               |
| Save Configuration        | `copy running-config startup-config`             |

## 📌 15. Common Mistakes to Avoid

| ❌ Mistake                               | ✅ Solution                              |
|------------------------------------------|------------------------------------------|
| Forgot to create VLAN before assigning   | Always create VLAN first                 |
| Trunk not configured between switches    | Configure trunk on BOTH sides            |
| VTP domain name mismatch                 | Use SAME domain name on all switches     |
| VTP password mismatch                    | Use SAME password on all switches        |
| Forgot to save config                    | Always run `copy run start`              |
| Wrong subnet on PC                       | Match PC subnet to its VLAN network      |
