# 🧠 VLAN (Virtual Local Area Network)

---

## 🔹 1. What is a VLAN?

- ==A **VLAN (Virtual Local Area Network)** is a **logical grouping** of devices on a switch, even if they are not on the same physical switch.==
- VLAN is a logical grouping of devices in a network, regardless of their physical location.

>  **Simple Definition:**
> VLAN divides **one physical switch** into **multiple virtual switches**.

**Key Rule:**

- 1 VLAN = 1 Broadcast Domain = 1 Subnet.
- Traffic from VLAN 10 cannot go to VLAN 20 without a Router (Layer 3) or (Layer 3 switch)

**How it Works**
- Devices are assigned to a VLAN
- Each VLAN acts like a separate network

### ⭐ Why Use VLAN?

| Benefit         | Description                                     |
| --------------- | ----------------------------------------------- |
| **Security**    | Departments can't access each other's traffic   |
| **Performance** | Reduces broadcast traffic                       |
| **Management**  | Easy to manage network per department           |
| **Cost Saving** | No need to buy separate switches per department |

__________


## 🔹 Types of VLAN

### 1. Default VLAN (VLAN 1)
 **Definition:** All switch ports belong to VLAN 1 by default

- Exists on every switch
- Cannot be deleted
- Carries control traffic (CDP, VTP, etc.)

❗ Important:
👉 Not secure → avoid using for real data

### 2. Data VLAN
**Definition:** VLAN used for normal user traffic

- PCs, laptops, printers
- Carries actual business data

**Example:**
- VLAN 10 → Employees
- VLAN 20 → Students


### 3. Voice VLAN
**Definition:** VLAN dedicated for IP phone traffic

- Used with VoIP devices
- Supports QoS (Quality of Service)

**Why separate?**
✔️ Voice needs low delay
✔️ Better call quality

**Example**
- VLAN 30 → IP Phones


### 4. Management VLAN
**Definition:** VLAN used to manage network devices

- SSH, Telnet, SNMP
- Only admins can access

**Why important?**
- Security (separate from user traffic)

- Example
- VLAN 99 → Network admin access


### 5. Native VLAN (Very Important 🔥)
**Definition:** VLAN that carries UNTAGGED traffic on a trunk port

**Key Concept**

**Normally:**
🔸 Trunk port → Tagged traffic (802.1Q)

**But:**
🔸 Native VLAN → No tag (untagged)
   
**Trunk Link**

```
[VLAN 10] → Tagged ✔  
[VLAN 20] → Tagged ✔  
[Native VLAN] → ❌ Untagged
```


**Need:**
- For compatibility with older devices
- Some protocols send untagged traffic

## 🔷 5. Types of VLAN Ports

### 🔸 Access Port

An **Access Port** is a switch port that carries traffic for **only one VLAN**.

It is used to connect **end devices**, such as:
- PC
- Laptop
- Printer
- IP Phone

- Connects to **end devices** (PC, Printer, Server)
- Carries traffic of **only ONE VLAN**
- Switch **adds/removes** VLAN tag automatically

### 🔸 Trunk Port
A **Trunk Port** carries **multiple VLANs over a single link**.

It is used to connect:

- Switch → Switch
- Switch → Router
- Switch → Server (multi VLAN)

To keep VLAN traffic separate, the switch **adds a VLAN tag (802.1Q)** to frames.

- Connects **switch to switch** (or switch to router)
- Carries traffic of **MULTIPLE VLANs**
- Uses **802.1Q tagging** to identify VLAN traffic

```
   ACCESS PORT                           TRUNK PORT
   ────────────                          ──────────

   ┌────┐    ┌──────┐                  ┌──────┐    ┌──────┐

## 🔷VLAN Header (802.1Q Tag)
When frames travel on a trunk link, they need a VLAN tag to identify which VLAN they belong to.

```
┌──────────────────────────────────────────────────────────────────────────┐
│                     802.1Q TAGGED FRAME                                  │
├────────────┬────────────┬──────────────┬──────────┬───────────┬─────────┤
│  Dest MAC  │  Src MAC   │  802.1Q Tag  │  Type    │   Data    │   FCS   │
│  (6 bytes) │  (6 bytes) │  (4 bytes)   │ (2 bytes)│ (46-1500) │(4 bytes)│
└────────────┴────────────┴──────┬───────┴──────────┴───────────┴─────────┘
                                 │
                    ┌────────────┴────────────┐
                    │     802.1Q TAG DETAIL   │
                    └────────────┬────────────┘
                                 ▼
┌─────────────────────────────────────────────────────────────────────────┐
│                        4-BYTE 802.1Q TAG                                │
├─────────────────────┬─────────────────────────────────────────────────┤
│        TPID         │                   TCI                            │
│     (16 bits)       │               (16 bits)                          │
│      0x8100         │                                                  │
├─────────────────────┼───────────┬─────────────┬───────────────────────┤
│                     │  Priority │     CFI     │       VLAN ID         │
│     Tag Protocol    │  (3 bits) │   (1 bit)   │      (12 bits)        │
│     Identifier      │   PCP     │   DEI       │     1 - 4094          │
└─────────────────────┴───────────┴─────────────┴───────────────────────┘
```

### 9. Trunk Port Configuration

When we have **2 or more switches** and VLANs must work across them.

```
  SWITCH 1 (SW1)                              SWITCH 2 (SW2)
  ┌──────────────────────┐                    ┌──────────────────────┐
  │                      │   TRUNK LINK       │                      │
  │ Fa0/1  → VLAN 10    │   (Fa0/24)         │ Fa0/1  → VLAN 10    │
  │ Fa0/3  → VLAN 20    │══════════════════   │ Fa0/3  → VLAN 20    │
  │ Fa0/5  → VLAN 30    │   carries ALL       │ Fa0/5  → VLAN 30    │
  │                      │   VLANs (10,20,30) │                      │
  └──────────────────────┘                    └──────────────────────┘

  PC1(Sales)  PC3(Market)  PC5(HR)           PC2(Sales) PC4(Market) PC6(HR)
```

##### Configure Trunk on SW1

```
SW1(config)# interface fa0/24
SW1(config-if)# switchport mode trunk
SW1(config-if)# switchport trunk allowed vlan 10,20,30
SW1(config-if)# exit
```

#####  Configure Trunk on SW2

```
SW2(config)# interface fa0/24
SW2(config-if)# switchport mode trunk
SW2(config-if)# switchport trunk allowed vlan 10,20,30
SW2(config-if)# exit
```

> 💡 If you want **ALL VLANs** to pass through trunk:
> ```
> SW1(config-if)# switchport trunk allowed vlan all
> ```

# 🧠 10. VTP (VLAN Trunking Protocol)

- VTP **automatically shares VLAN information** between switches so you don't have to create VLANs on every switch manually.
- ==VTP is a protocol used to automatically share VLAN information between switches.==

```
  ┌────────────┐   TRUNK    ┌────────────┐   TRUNK    ┌────────────┐
  │   SW1      │═══════════ │    SW2     │═══════════ │    SW3     │
  │ Layer 2    │            │  Layer 2   │            │  Layer 2   │
  │ VTP SERVER │            │ VTP CLIENT │            │ VTP CLIENT │
  └────────────┘            └────────────┘            └────────────┘
       │                         │                         │
  Create VLAN 10,20,30    Automatically              Automatically
  HERE                    gets VLAN 10,20,30         gets VLAN 10,20,30
                          via TRUNK                  via TRUNK

  ✅ ALL Layer 2 switches — NO Layer 3 needed!
```

```
┌────────────────────────────────────────────────────────────────────────────┐
│                           VTP OVERVIEW                                     │
├────────────────────────────────────────────────────────────────────────────┤
│                                                                            │
│  VTP = VLAN Trunking Protocol                                              │
│                                                                            │
│  PURPOSE: Automatically synchronize VLAN information across all switches   │
│                                                                            │
│  ┌────────────────────────────────────────────────────────────────────────┐│
│  │  WITHOUT VTP:                                                          ││
│  │  ─────────────                                                         ││
│  │    • Create VLAN 10 on Switch 1                                        ││
│  │    • Manually create VLAN 10 on Switch 2                               ││
│  │    • Manually create VLAN 10 on Switch 3                               ││
│  │    • Manually create VLAN 10 on Switch 4...                            ││
│  │    • 100 switches = 100 times manual configuration! 😫                 ││
│  │                                                                         ││
│  ├────────────────────────────────────────────────────────────────────────┤│
│  │  WITH VTP:                                                             ││
│  │  ──────────                                                            ││
│  │    • Create VLAN 10 on VTP Server                                      ││
│  │    • ALL switches automatically learn VLAN 10! ✅                      ││
│  │    • 100 switches = 1 time configuration! 😊                           ││
│  └────────────────────────────────────────────────────────────────────────┘│
│                                                                            │
│  CISCO PROPRIETARY: Works only on Cisco switches                           │
│  LAYER 2 PROTOCOL: Works over trunk links                                  │
│                                                                            │
└────────────────────────────────────────────────────────────────────────────┘
```

**VTP is Used to create multiple VLAN (1-1000)**
**Its Require Layer 3 switch which has 64 Ports so we can connect 64 layer 2 switch and Layer 2 switch has 24 ports to connect n device this how we can create 1000 VLAN.**
==We have to configure VLANs on Layer 3 switch only or Layer 2 Switch but we have to make trunk port between switch.==

### VTP Modes
==VTP modes define how a switch behaves in sharing or handling VLAN information==


**Why Do We Need VTP Modes?**

Because not all switches should have same control

- Some should control VLANs
- Some should just follow
- Some should stay independent


 ####  1. **Server**
**Definition:** Main switch that controls VLANs

- Can create, modify, delete VLANs
- Sends updates to other switches

#### 2. Client
**Definition:** Receives VLAN info from server / Follower switch

Cannot create VLANs

#### 3. Transparent
**Definition**: Does NOT participate in VTP / Independent switch

 - Forwards VTP messages
 - Can create its own VLANs locally
 - Does NOT follow VTP


| Mode            | Can Create VLAN? | Learns VLANs? | Forwards VTP? |
| --------------- | ---------------- | ------------- | ------------- |
| **Server**      | ✅ Yes            | ✅ Yes         | ✅ Yes         |
| **Client**      | ❌ No             | ✅ Yes         | ✅ Yes         |
| **Transparent** | ✅ Yes (local)    | ❌ No          | ✅ Yes         |

## 13. Quick Command Reference

| Task                 | Command                                                |
| -------------------- | ------------------------------------------------------ |
| Create VLAN          | `vlan 10` → `name SALES`                               |
| Show VLANs           | `show vlan brief`                                      |
| Assign Access Port   | `switchport mode access` → `switchport access vlan 10` |
| Configure Trunk      | `switchport mode trunk`                                |
| Allow VLANs on Trunk | `switchport trunk allowed vlan 10,20,30`               |
| Show Trunk           | `show interfaces trunk`                                |
| Set VTP Mode         | `vtp mode server/client/transparent`                   |
| Set VTP Domain       | `vtp domain MYCOMPANY`                                 |
| Set VTP Password     | `vtp password cisco123`                                |
| Show VTP Status      | `show vtp status`                                      |
| Delete All VLANs     | `delete flash:vlan.dat` → `reload`                     |
| Save Configuration   | `copy running-config startup-config`                   |

##  15. Common Mistakes to Avoid

| ❌ Mistake                              | ✅ Solution                           |
| -------------------------------------- | ------------------------------------ |
| Forgot to create VLAN before assigning | Always create VLAN first             |
| Trunk not configured between switches  | Configure trunk on BOTH sides        |
| VTP domain name mismatch               | Use SAME domain name on all switches |
| VTP password mismatch                  | Use SAME password on all switches    |
| Forgot to save config                  | Always run `copy run start`          |
| Wrong subnet on PC                     | Match PC subnet to its VLAN network  |
