
# Address Resolution Protocol(ARP)
==ARP is used to find the MAC address of a device when its IP address is known.==
- Maps an IP address to a MAC address on the same local network.
- ARP is used to find the MAC address of a device from its IP address in a local network.

**Who maintains ARP table**

👉 Any device with IP + MAC address 
- PC 
- Laptop 
- Router 
- Switch (Layer 3) 

**Who generates ARP request?**

👉 Any device that wants to find a MAC address 
- PC 
- Router 
- Server 
- Any host 


## Working
**example:**
PC1 (192.168.1.10) wants to send data to PC2 (192.168.1.20) on the same LAN.

#### 1: Check ARP Cache
PC1 checks its local ARP table to see if it already knows PC2’s MAC address
```bash
PC1> arp -a
# If not found → proceed to ARP Request
```

#### 2: Send ARP Request (Broadcast)
PC1 broadcasts an ARP Request frame to all devices on the LAN:

```
┌─────────────────────────────────────────────┐
│  Who has 192.168.1.20? Tell 192.168.1.10    │
│  Sender MAC: AA:AA:AA:AA:AA:AA              │
│  Sender IP: 192.168.1.10                    │
│  Target IP: 192.168.1.20                    │
│  Target MAC: 00:00:00:00:00:00 (unknown)    │
└─────────────────────────────────────────────┘
► Destination MAC: FF:FF:FF:FF:FF:FF (Broadcast)
```

==MAC: FF:FF:FF:FF:FF:FF (Broadcast)==
#### 3: Target Device Replies (Unicast)
Only PC2 (with IP 192.168.1.20) responds with an ARP Reply:

```
┌─────────────────────────────────────────────┐
│  192.168.1.20 is at BB:BB:BB:BB:BB:BB      │
│  Sender MAC: BB:BB:BB:BB:BB:BB              │
│  Sender IP: 192.168.1.20                    │
│  Target MAC: AA:AA:AA:AA:AA:AA              │
│  Target IP: 192.168.1.10                    │
└─────────────────────────────────────────────┘
► Destination MAC: AA:AA:AA:AA:AA:AA (Unicast)
```

####  4: Update ARP Tables & Communicate
- PC1 adds PC2’s IP→MAC mapping to its ARP cache
- Now PC1 can send frames directly to PC2 using its MAC address
- Communication proceeds via unicast

==ARP table Reset after every 4 hours at Router== 

## Key Concepts
1. ARP is Local Only
ARP only works within the same broadcast domain (same VLAN/subnet). It cannot cross routers.
      ==To communicate with devices outside your subnet, you use the default gateway’s MAC, not the destination device’s MAC.==

## Types of ARP

### 1. Standard ARP (Normal ARP)

Used to **find the MAC address when the IP address is known**.

**Example**

```
Who has 192.168.1.20 ?
Tell 192.168.1.10
```

Response:

```
192.168.1.20 is at AA:BB:CC:DD:EE:FF
```

Used in **normal LAN communication**.

---

### 2. Reverse ARP (RARP)

Used to **find the IP address when the MAC address is known**.

**Example**

```
MAC → Request → Get IP
```

Used by **old diskless devices** to obtain an IP address.

⚠️ Mostly replaced by **DHCP**.

---

### 3. Proxy ARP

A **router responds to an ARP request on behalf of another device**.

**Example**

```
PC1 → Who has 192.168.2.10 ?
Router → I have it
```

Router then forwards the packet to the real destination.

Used when **devices are on different networks but appear in the same network**.

---

### 4. Gratuitous ARP

A device **broadcasts its own IP–MAC mapping without being asked**.

**Example**

```
192.168.1.10 is at AA:BB:CC:DD:EE:FF
```

Used for:

- Detecting **duplicate IP addresses**
- **Updating ARP tables**
- **Failover systems**

---

## Summary

| Type | Purpose |
|-----|-----|
| Standard ARP | IP → MAC resolution |
| RARP | MAC → IP resolution |
| Proxy ARP | Router replies for another device |
| Gratuitous ARP | Device announces its own IP-MAC |

---

# Default Gateway 
==A default gateway is the device (usually a router) that sends data from your network to other networks (like the internet).==

The default gateway is the IP address of the router that a device uses to send traffic when the destination IP is not on the local network (i.e., outside the local subnet).


```
┌──────────────────────────────────────────────────────┐
│  "If the destination isn't on my local network,      │
│   send it to the default gateway!"                   │
└──────────────────────────────────────────────────────┘
```

```
┌──────────────────────────────────────────────────────────────────┐
│                     LOCAL NETWORK                                │
│                    192.168.1.0/24                                │
│                                                                  │
│   ┌─────────┐       ┌─────────┐       ┌─────────┐               │
│   │   PC1   │       │   PC2   │       │  Printer│               │
│   │.1.10    │       │.1.20    │       │.1.30    │               │
│   └────┬────┘       └────┬────┘       └────┬────┘               │
│        │                 │                 │                    │
│        └─────────────────┼─────────────────┘                    │
│                          │                                      │
│                    ┌─────┴─────┐                                │
│                    │  Switch   │                                │
│                    └─────┬─────┘                                │
│                          │                                      │
│                    ┌─────┴─────┐                                │
│                    │  Router   │  ← DEFAULT GATEWAY             │
│                    │ 192.168.1.1│                               │
│                    └─────┬─────┘                                │
└──────────────────────────┼──────────────────────────────────────┘
                           │
                           ▼
                    ┌─────────────┐
                    │  INTERNET   │
                    │  10.0.0.0/8 │
                    │ 172.16.0.0  │
                    └─────────────┘
```

- 192.168.1.50 → Same subnet → sends directly (uses ARP to find MAC)
- 8.8.8.8 (Google) → Different subnet → sends to default gateway (192.168.1.1)


## 🌐 Default Gateway Packet Flow

👉 Definition: Process of sending data from a host to a remote network via a router (default gateway)

---

## 🔹 Host Information

🔸 Host (PC)
    IP: 192.168.1.10/24

🔸 Default Gateway (Router)
    IP: 192.168.1.1

🔸 Destination Host
    IP: 172.16.10.5

---

## 🔹 1. Host wants to send a packet

👉 Host creates packet  
    Destination IP: 172.16.10.5

---

## 🔹 2. Host checks subnet

👉 Host Network: 192.168.1.0/24  
👉 Destination: 172.16.10.5  

❌ Destination is NOT in same network  
👉 Remote network

---

## 🔹 3. Host decides next hop

👉 Next Hop = Default Gateway (Router)  
    192.168.1.1

---

## 🔹 4. Host needs MAC of Router

👉 If MAC not in ARP table:

🔸 Host sends ARP Request  
    "Who has 192.168.1.1 ?"

🔸 Router replies (ARP Reply)  
    "192.168.1.1 is at 00:AA:BB:CC:DD:EE"

---

## 🔹 5. Host builds Ethernet Frame

👉 Ethernet Header
    Source MAC: Host (PC) MAC  
    Destination MAC: Router MAC  

👉 IP Header
    Source IP: 192.168.1.10  
    Destination IP: 172.16.10.5  

👉 Frame sent from **Host → Router**

---

## 🔹 6. Router receives packet

👉 Router removes Ethernet frame  
👉 Router checks routing table  

👉 Router creates NEW Ethernet Frame:

👉 Ethernet Header
    Source MAC: Router Interface MAC  
    Destination MAC: Next Hop MAC  

👉 IP Header (UNCHANGED)
    Source IP: 192.168.1.10  
    Destination IP: 172.16.10.5  

👉 Packet forwarded toward destination

---

## 🔁 Key Concepts

👉 Host = device sending data (PC)  
👉 Router = default gateway (next hop device)  

👉 MAC changes at every hop  
👉 IP remains same end-to-end  

---

## ⚡ Easy Trick

👉 Host → sends to Router MAC  
👉 Router → forwards using routing table

---

## Key Networking Rule

```
MAC address = changes at every hop
IP address  = remains the same end-to-end
```

# Default Route
- A default route is a special routing table entry that tells a router where to send packets when there is no specific route to the destination network.

==Default Route = "If no route matches, send packet here"==

- A **Default Route** is a routing rule used by routers to **forward packets when no specific route to the destination network exists**.
- It acts as a **catch-all route**.

```
┌──────────────────────────────────────────────────────┐
│  "If I don't know where this packet goes,            │
│   send it to the default route!"                     │
└──────────────────────────────────────────────────────┘
```

==Represented as: 0.0.0.0/0 or 0.0.0.0 0.0.0.0 (matches ALL destinations)==

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   PC (End Device)                    Router                    │
│   ┌──────────────┐                  ┌──────────────┐           │
│   │ Default      │                  │ Default      │           │
│   │ Gateway:     │────sends to─────►│ Route:       │           │
│   │ 192.168.1.1  │                  │ 0.0.0.0/0    │           │
│   └──────────────┘                  └──────────────┘           │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### example of working
#### 1: Router Receives Packet
```
Router receives packet from PC1:
Source: 192.168.1.10
Destination: 8.8.8.8
```

####  2: Router Checks Routing Table
```
Router performs longest prefix match lookup:

Route Table:
C  192.168.1.0/24 → Directly connected      ❌ (8.8.8.8 doesn't match)
S* 0.0.0.0/0 → via 203.0.113.1              ✅ (Matches everything!)
```

#### 3: Router Forwards to Next-Hop
```
Router forwards packet to 203.0.113.1 (ISP)
ISP routes it toward 8.8.8.8
```

#### 4: Return Traffic
```
Reply from 8.8.8.8 comes back through ISP
Router receives it, checks routing table
Matches 192.168.1.0/24 (directly connected)
Delivers to PC1
```

# ICMP (Internet Control Message Protocol)

ICMP is a Layer 3 protocol (works directly over IP) whose only job is to report errors, diagnostics, and control information about IP packet processing.

- ==**ICMP** is a **network layer protocol used by devices to send error messages and network status information.**==
- It helps diagnose network problems.

It does NOT carry user data like HTTP, FTP, or SSH. It is purely a control/support protocol.

ICMP (Internet Control Message Protocol) is a network layer protocol used for:
- Reporting errors in IP packet processing
- Diagnostic functions (ping, traceroute)
- Network troubleshooting and management

### Message Format
```
┌──────────┬──────────┬────────────────┐
│   Type   │   Code   │    Checksum    │
│ (8 bits) │ (8 bits) │   (16 bits)    │
├──────────┴──────────┴────────────────┤
│              Data                    │
└──────────────────────────────────────┘
```


- Type = What kind of message
- Code = More details about the message
- Checksum = Error checking

#### TTL is a value inside the IP header.

To stop packets from looping forever.

**Every router hop:** decreases TTL by 1

If TTL becomes 0: router drops the packet
and sends back an ICMP message.

**TTL in windows vs linux:**
Linux usually starts with 64
Windows usually starts with 128

### Two Category
```
ICMP Messages
      │
      ├── Error Messages (Something went wrong)
      │     • Destination Unreachable
      │     • Time Exceeded
      │
      └── Query Messages (Asking/Responding)
            • Ping Request
            • Ping Reply
```
