# Collision Domain

## Definition
A **collision domain** is a part of a network where **data packets can collide with each other when multiple devices transmit at the same time**.

Collisions happen when **two devices send data on the same shared medium simultaneously**.

---

# Why Collision Happens

In older Ethernet networks, devices shared the **same communication medium**.

Example:

```
PC1 ----\
PC2 ----- Hub ----- PC3
PC4 ----/
```

If **PC1 and PC3 send data at the same time**, their signals collide on the network.

This is called a **collision**.

---

# How Collision is Handled

Ethernet uses **CSMA/CD (Carrier Sense Multiple Access with Collision Detection)**.

Steps:

1. Device checks if medium is free
2. If free → send data
3. If collision happens → stop transmission
4. Wait random time
5. Retransmit data

---

# Devices and Collision Domain

| Device | Collision Domain Behavior                     |
| ------ | --------------------------------------------- |
| Hub    | One collision domain for all devices          |
| Switch | Each port is a separate collision domain      |
| Router | Each interface is a separate collision domain |

---

# Example

## Hub Network

```
PC1
   \
PC2 --- Hub --- PC3
   /
PC4
```

All devices share **one collision domain**.

So collisions are **very common**.

---

## Switch Network

```
PC1 ──\
PC2 ─── Switch ─── PC3
PC4 ──/
```

Each port has its **own collision domain**.

So collisions **do not occur between ports**.

---

# Broadcast Domain vs Collision Domain

| Feature           | Broadcast Domain                    | Collision Domain                   |
| ----------------- | ----------------------------------- | ---------------------------------- |
| Meaning           | Devices receiving broadcast traffic | Devices where collisions can occur |
| Broken by         | Router / VLAN                       | Switch / Router                    |
| Broadcast Address | FF:FF:FF:FF:FF:FF                   | Not related                        |

---

# Quick Memory Trick

```
Hub → 1 Collision Domain
Switch → 1 Collision Domain per Port
Router → 1 Collision Domain per Interface
```




# Broadcast Domain

A broadcast domain is a group of devices in a network that receive the same broadcast message sent by any device in that network.

**Simple Definition**

A broadcast domain is a part of a network where broadcast traffic is forwarded to all devices.

### Example

If a PC sends a broadcast packet like:

```
Destination MAC: FF:FF:FF:FF:FF:FF
```

Then all devices in that broadcast domain receive the packet.



#### Example Network:
```
PC1 ──┐
PC2 ──┤
PC3 ──┤── Switch ── Router
PC4 ──┘
```

**Here:**
- PC1 sends a broadcast
- Switch forwards it to PC2, PC3, PC4
- Router does NOT forward broadcast

So all PCs connected to the switch are in one broadcast domain.


#### Key Points

- Broadcast uses MAC address:
		```
		FF:FF:FF:FF:FF:FF
		```

- Switch does NOT break broadcast domains
- Router breaks broadcast domains


#### Devices and Broadcast Domain

| Device | Broadcast Domain Behavior          |
| ------ | ---------------------------------- |
| Hub    | Same broadcast domain              |
| Switch | Same broadcast domain              |
| Router | Creates separate broadcast domains |
| VLAN   | Creates separate broadcast domains |


### Example with Router

```
Network 1            Network 2

PC1 ─ Switch ─ Router ─ Switch ─ PC2
```

Broadcast from PC1 will NOT reach PC2 because the router blocks broadcast traffic.

**So:**
```
Network 1 → Broadcast Domain 1
Network 2 → Broadcast Domain 2
```

#### 1. Switch → Same Broadcast Domain
#### 2. Router → Breaks Broadcast Domain
#### 3. VLAN → Creates Multiple Broadcast Domains

# 2. HUB (Layer 1 - Physical Layer)
A Hub is a basic networking device that connects multiple Ethernet devices together, making them act as a single network segment.

==A **hub** is a **Layer 1 (Physical Layer)** device that connects multiple devices in a network and **broadcasts incoming data to all ports**.==
- When a device sends data to the hub
- The hub forwards the data to **all connected devices**
- **Security view:** Very insecure, easy sniffing.

## Working:
- **Broadcasting Device:** When a hub receives data on one port, it floods/broadcasts it to ALL other ports
- **No Intelligence:** Cannot read MAC addresses or make forwarding decisions
- **Half-Duplex:** Only one device can transmit at a time
- **Collision Domain:** All connected devices share ONE collision domain
- One **broadcast domain**

# 3. BRIDGE

A bridge is a Layer 2 (Data Link Layer) device that connects two or more network segments and filters traffic based on MAC addresses.

==A **bridge** is a **Layer 2 device** used to **connect two LAN segments** and filter traffic using **MAC addresses**.==


- **Purpose:** Connects two LAN segments and filters using MAC.
- **Layer:** OSI Layer 2 (Data Link)
- **Key point:** Reduces collisions.
- **Security view:** Better than hub, but still limited.
- **Broadcast Domain:** Single Broadcast Domain
- **Spanning Tree Protocol (STP)** - Preventing Loops



![[bridge.png]]


```
  Segment A              Segment B
[PC1]--[PC2]---[BRIDGE]---[PC3]--[PC4]
     
Collision Domain 1  |  Collision Domain 2
```

### MAC Table (Store MAC table )

**Frame Forwarding Decision**
```
DECISION FLOWCHART:

Frame arrives on Port 1, Destination = CC
        ↓
Is CC in MAC table?
        ↓
    [YES] → CC on Port 2 → Forward to Port 2 ONLY
        
    [NO] → FLOOD to all ports except source
```

# 4. SWITCH (Layer 2 - Data Link Layer)
==A **switch** is a **Layer 2 (Data Link Layer)** device that connects devices in a LAN and forwards frames traffic using **MAC addresses**.==

- A Switch is an advanced multi-port bridge that operates at Layer 2 (or Layer 3 for multilayer switches). It's the backbone of modern LANs, providing dedicated bandwidth to each port.

- **Purpose:** Sends frames only to the correct port using MAC table.
- **Layer:** OSI Layer 2 (Data Link)
- **Key points:**
	  - Uses MAC address table (CAM table)
	  - Each port is separate collision domain
- Same **broadcast domain** (unless VLANs used)
- Full duplex communication

## MAC Address Table (CAM Table) 
A MAC table is a table inside a switch that stores MAC addresses of devices and the ports they are connected to.

The switch uses this table to forward frames to the correct device instead of broadcasting to all ports.

Example MAC Table
```
MAC Address        Port
-------------------------
AA:11:22:33:44:01  Fa0/1
AA:11:22:33:44:02  Fa0/2
AA:11:22:33:44:03  Fa0/3
```

```
Device MAC AA:11:22:33:44:01 → connected to port Fa0/1
```

#  ==5. ROUTER (Layer 3 Device)==
==A **router** is a **Layer 3 (Network Layer)** device that connects **different networks** and forwards packets using **IP addresses** and **Routing Table**.==

- A Router is a Layer 3 device that interconnects different networks and makes forwarding decisions based on IP addresses. It's the fundamental device for internetworking and WAN connectivity.


- **Purpose:** Connects different networks and routes using IP.
- **Layer:** OSI Layer 3 (Network)
- **Key points:**
  - Uses routing table
  - Breaks broadcast domains
  - Each router interface = separate broadcast domain
  - Each router interface = separate collision domain


A Router is a network device operating at the Network Layer (Layer 3) that performs packet switching based on logical addressing (IP), maintains routing information bases (RIB), and executes routing protocols to determine optimal paths for data transmission across interconnected networks.

```
PACKET JOURNEY THROUGH ROUTER:

Step 1: Packet arrives on interface
        ┌─────────────────────────────────────────┐
        │ Ethernet Frame                          │
        │ ┌─────────────────────────────────────┐ │
        │ │ Dest MAC │ Src MAC │ IP Packet     │ │
        │ └─────────────────────────────────────┘ │
        └─────────────────────────────────────────┘

Step 2: Router checks destination MAC
        • If matches router's interface MAC → Accept
        • If doesn't match → Discard

Step 3: Router strips Layer 2 header, examines IP header
        ┌─────────────────────────────────────┐
        │ IP Packet                           │
        │ Src IP: 10.1.1.10                   │
        │ Dst IP: 10.2.2.10  ← Router reads   │
        └─────────────────────────────────────┘

Step 4: Routing table lookup for destination
        Router checks: "How do I reach 10.2.2.10?"

Step 5: Determine exit interface and next-hop

Step 6: Decrement TTL (Time to Live)
        • If TTL = 0 → Discard, send ICMP Time Exceeded
        • If TTL > 0 → Continue

Step 7: Build new Layer 2 header
        • Source MAC = Router's exit interface MAC
        • Dest MAC = Next-hop device MAC (ARP lookup)

Step 8: Forward packet out exit interface
```

## 7) Modem (Modulator–Demodulator)
A modem is a device that connects your home/office network to the Internet (ISP).

```
Laptop → Router → Modem → ISP → Internet
```

**It converts signals between:**
- Digital (your devices)
- Analog / ISP signal (cable, DSL, fiber)

**How It Works**
- Your computer sends digital data (0s & 1s)
- Modem converts it → signal suitable for ISP line (modulation)
- Incoming signal → converted back to digital (demodulation)



## 9) Load Balancer

A load balancer distributes incoming traffic across multiple servers.

```
User → Load Balancer → Multiple Servers ✅
```

**Goal:**
- Prevent overload
- Improve performance
- Ensure availability
- **Layer:** Layer 4 or Layer 7

### Types of Load Balancing
#### 🔹 Round Robin
Requests distributed one by one
#### 🔹 Least Connections
Sends traffic to least busy server
#### 🔹 IP Hash
Same user always goes to same server

### Types by Layer
- Layer 4 (Transport) → based on IP/Port
- Layer 7 (Application) → based on URL, HTTP
