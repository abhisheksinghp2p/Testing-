# Firewall

- A firewall is a network security device/software that monitors and controls incoming and outgoing network traffic based on predetermined security rules.
- ==A firewall is a security system that monitors and controls incoming and outgoing network traffic based on rules.==

**Purpose**
- Block unauthorized access
- Allow trusted traffic
- Protect network
  
```
[Internet] ←→ [FIREWALL] ←→ [Internal Network]
                  ↓
         Filters traffic based
         on security rules
```

## Types of Firewalls / stateless
#### 1. Packet Filtering Firewall
- Works mainly on **IP + Port + Protocol**
- Fast, basic filtering
- Usually **stateless**
- Example: basic router ACLs

```
┌─────────────────────────────────────┐
│   Examines packets at Network Layer │
│   (Layer 3 - OSI Model)             │
├─────────────────────────────────────┤
│ Checks:                             │
│ • Source IP Address                 │
│ • Destination IP Address            │
│ • Source Port                       │
│ • Destination Port                  │
│ • Protocol (TCP/UDP/ICMP)           │
└─────────────────────────────────────┘
```

**Pros:** Fast, low resource usage
**Cons:** No deep inspection, vulnerable to IP spoofing

#### 2. Stateful Inspection Firewall
Check inside packet Payload/Signature.

- Tracks active connections using a **state table**
- Can allow return traffic automatically
- More secure than packet filtering
- Common in enterprise firewalls

> [!NOTE]
> **DPI:** Deep Packet Inspection.

```
┌────────────────────────────────────────┐
│   Tracks active connection states      │
│   Operates at Layer 3-4                │
├────────────────────────────────────────┤
│ State Table Example:                   │
│ ┌────────────┬──────┬───────┬───────┐ │
│ │ Source IP  │ Port │ Dest  │ State │ │
│ ├────────────┼──────┼───────┼───────┤ │
│ │ 192.168.1.5│ 5050 │ 80    │ ESTAB │ │
│ │ 10.0.0.2   │ 3389 │ 443   │ NEW   │ │
│ └────────────┴──────┴───────┴───────┘ │
└────────────────────────────────────────┘
```

#### 3. Application Layer(1) Firewall (Proxy Firewall)

- Firewall acts as an **intermediate**
- Works at **Layer 7**
- Can inspect application content (HTTP, FTP, etc.)
- Higher security but slower
```
Client → [Proxy Firewall] → Server
              ↓
    Inspects application data
    (HTTP, FTP, DNS, SMTP)
```

**Features:**
- Deep packet inspection
- Content filtering
- User authentication
- Caching capabilities

##### Proxy Firewall 
act as middle man b/w user and server ensuring a direct connection with external server never made

### 4. Next-Generation Firewall (NGFW)
- Combines firewall + advanced security features
- Features include:
  - Deep Packet Inspection (DPI)
  - Application awareness (App control)
  - IPS/IDS
  - URL filtering
  - Malware detection
```
┌─────────────────────────────────────────┐

│ ✓ SSL/TLS Inspection                    │

│ ✓ Sandboxing                            │
│ ✓ Identity-based policies               │
└─────────────────────────────────────────┘
```


#### Other Firewalls
| Type                               | Description                          | Deployment                        | Use Case                         |
| ---------------------------------- | ------------------------------------ | --------------------------------- | -------------------------------- |
| **Network Firewall**               | Protects entire network at perimeter | Hardware/Software at network edge | Enterprise network protection    |
| **Host-based Firewall**            | Protects individual devices          | Software on endpoints             | Personal computers, servers      |
| **Cloud Firewall (FWaaS)**         | Cloud-based security service         | Cloud infrastructure              | Cloud workloads, SaaS protection |
| **WAF (Web Application Firewall)** | Protects web applications            | In front of web servers           | Web apps, APIs protection        |
| **Virtual Firewall**               | For virtualized environments         | Hypervisor/VM level               | Data centers, virtual networks   |

## Proxy Servers/Firewall
```
┌─────────────────────────────────────────────────────────┐
│                    PROXY SERVER                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  DEFINITION:                                            │
│  An intermediary server that sits between client        │
│  devices and destination servers. It receives           │
│  requests from clients, forwards them to servers,       │

│                                                         │
│  BASIC FUNCTION:                                        │
│  Client → Proxy → Internet/Server → Proxy → Client      │
│                                                         │

│  • Privacy & Anonymity                                  │
│  • Caching & Performance                                │
│  • Access Control & Filtering                           │
│  • Security & Monitoring                                │
│  • Load Balancing                                       │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### Types of Proxy Servers
#### 1. Forward Proxy
```
┌─────────────────────────────────────────────────────────┐
│                    FORWARD PROXY                        │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  DEFINITION:                                            │
│  A proxy server that sits in front of CLIENT machines   │
│  and forwards client requests to the internet.          │
│  The server sees the proxy's IP, not the client's.      │
│                                                         │
│  USE CASES:                                             │
│  • Corporate internet access control                    │
│  • Content filtering (block social media)               │
│  • Bandwidth optimization (caching)                     │
│  • User anonymity                                       │
│  • Bypassing geo-restrictions                           │
│                                                         │
│  WHO USES IT:                                           │
│  Clients/Users to access internet                       │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

#### 2. Reverse Proxy
```
┌─────────────────────────────────────────────────────────┐
│                    REVERSE PROXY                        │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  DEFINITION:                                            │
│  A proxy server that sits in front of SERVER machines   │
│  and forwards client requests to appropriate backend    │
│  servers. The client sees only the proxy, not the       │
│  actual servers behind it.                              │
│                                                         │
│  USE CASES:                                             │
│  • Load balancing across multiple servers               │
│  • SSL/TLS termination                                  │
│  • Caching static content                               │
│  • DDoS protection                                      │
│  • Hiding backend infrastructure                        │
│  • Web acceleration                                     │
│                                                         │
│  WHO USES IT:                                           │
│  Server administrators to protect/optimize servers      │
│                                                         │
│  EXAMPLES:                                              │
│  Nginx, HAProxy, Apache (mod_proxy), Cloudflare         │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

#### 3. Transparent Proxy
```
┌─────────────────────────────────────────────────────────┐
│                  TRANSPARENT PROXY                      │
│              (Intercepting Proxy)                       │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  DEFINITION:                                            │
│  A proxy that intercepts traffic without requiring      │
│  any client configuration. Users may not even know      │
│  their traffic is being proxied. Works at network       │
│  level through traffic redirection.                     │
│                                                         │
│  CHARACTERISTICS:                                       │
│  • No client configuration needed                       │
│  • Invisible to end users                               │
│  • Does NOT modify requests/responses                   │
│  • Client IP is visible to destination                  │
│                                                         │
│  USE CASES:                                             │
│  • ISP content filtering                                │
│  • Corporate monitoring                                 │
│  • Bandwidth management                                 │
│  • Content caching (CDN)                                │
│  • Parental controls                                    │
│                                                         │
│  EXAMPLES:                                              │
│  Squid (transparent mode), ISP proxies                  │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

| Proxy Type            | Direction         | Client Config | Hides Client IP | Server Knows Proxy | Primary Use               |
| --------------------- | ----------------- | ------------- | --------------- | ------------------ | ------------------------- |
| **Forward Proxy**     | Client → Internet | Required      | Yes             | Sometimes          | Access control, filtering |
| **Reverse Proxy**     | Internet → Server | Not needed    | N/A             | N/A                | Load balancing, security  |
| **Transparent Proxy** | Client → Internet | Not needed    | No              | Yes                | Caching, monitoring       |
| **Anonymous Proxy**   | Client → Internet | Required      | Yes             | Depends on level   | Privacy, anonymity        |

###  IDS - Intrusion Detection System
```
┌─────────────────────────────────────────────────────────────────┐
│                 IDS - INTRUSION DETECTION SYSTEM                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  DEFINITION:                                                    │
│  A security system that monitors network traffic or system      │
│  activities for malicious activities or policy violations       │
│  and ALERTS administrators. It is PASSIVE - detection only.     │
│                                                                 │
│  KEY CHARACTERISTIC:                                            │
│  ┌─────────────────────────────────────────────────┐            │
│  │     DETECTS  ✓     but    DOES NOT BLOCK  ✗     │           │
│  └─────────────────────────────────────────────────┘            │
│                                                                 │
│  TYPES:                                                         │
│  • NIDS (Network-based IDS) - Monitors network traffic          │
│  • HIDS (Host-based IDS) - Monitors individual systems          │
│                                                                 │
│  DETECTION METHODS:                                             │
│  • Signature-based - Matches known attack patterns              │
│  • Anomaly-based - Detects deviation from baseline              │
│  • Heuristic-based - Uses rules and AI                          │
│                                                                 │
│  EXAMPLES:                                                      │
│  Snort (open-source), Suricata, OSSEC, Zeek (Bro)               │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

```

# Virtual Private Network (VPN)
==VPN (Virtual Private Network) creates a secure, encrypted connection over a less secure network (typically the internet).==
VPN creates a secure encrypted tunnel over the internet to connect to another network.

```
WITHOUT VPN:
┌────────┐     [Open Internet]     ┌────────────┐
│  You   │ ──────────────────────► │  Website   │
└────────┘   Data visible to:      └────────────┘
             • ISP
             • Hackers
             • Government
             • Network Admin

WITH VPN:
┌────────┐    Encrypted Tunnel    ┌────────────┐     ┌────────────┐
│  You   │ ════════════════════►  │ VPN Server │ ──► │  Website   │
└────────┘                        └────────────┘     └────────────┘
             ISP sees only                Website sees
             encrypted data               VPN's IP
```

## Concepts
1. Tunneling

   Original Packet:

┌──────────────────────────────────────────────────────────┐
│ New IP │ VPN    │ ████████████████████████████████████  │
│ Header │ Header │ ██ ENCRYPTED ORIGINAL PACKET ███████  │
└──────────────────────────────────────────────────────────┘
```

## VPN Key Terms

| Term                          | Definition                              |
| ----------------------------- | --------------------------------------- |
| **Tunnel**                    | Encrypted pathway between two endpoints |
| **Endpoint**                  | Device/server where tunnel terminates   |
| **Encapsulation**             | Wrapping one protocol inside another    |
| **Cipher**                    | Algorithm used for encryption           |
| **Handshake**                 | Initial secure connection establishment |
| **SA (Security Association)** | Set of policies for the VPN connection  |

**From Networking Perspective:**
- Connect geographically separated networks
- Extend private network across public infrastructure
- Enable remote access to internal resources

**From Security Perspective:**
- Confidentiality: Encrypt data in transit
- Integrity: Prevent data tampering
- Authentication: Verify identity of endpoints
- Privacy: Hide source IP and traffic patterns
