Linux is multi-user. Every file/process belongs to a user and group.

# 1) Explore User/Group Files

1. **/etc/passwd** → identifies user
2. **/etc/shadow** → verifies password
3. **/etc/group** → assigns permissions

## 1. /etc/passwd - User database

**Basic information about all users**
- This file is readable by everyone
- Password is NOT stored here anymore (just x)

```bash
cat /etc/passwd
```

Format: username:x:UID:GID:comment:home:shell

**Example:**
```
john:x:1001:1001:John Doe:/home/john:/bin/bash
```

- john = username
- x = password (stored in /etc/shadow)
- 1001 = User ID
- 1001 = Primary Group ID
- /home/john = home directory
- /bin/bash = default shell


## 2. /etc/shadow - Password hashes (secure)

- Encrypted password **hashes + password policies**
- Only root can read
```bash
sudo cat /etc/shadow
```

```
ak:$6$randomhash:19000:0:99999:7:::
```

| Field  | Meaning                |
| ------ | ---------------------- |
| ak     | Username               |
| $6$... | Password hash          |
| 19000  | Last password change   |
| 0      | Min days before change |
| 99999  | Max days               |
| 7      | Warning days           |

## 3. /etc/group - Group database
Information about groups and their members

```bash
cat /etc/group
```

**example:**
```
sudo:x:27:ak
```

| Field | Meaning                |
| ----- | ---------------------- |
| sudo  | Group name             |
| x     | Password (rarely used) |
| 27    | Group ID               |
| ak    | Members                |
**What groups am I in?**
```
groups
```



## /etc/hosts 

`/etc/hosts` is a local DNS file

**It maps:**
	IP address → hostname

**Example:**
```
127.0.0.1   localhost
192.168.1.10   myserver
```


## Why use /etc/hosts?

- ✔ Testing servers
- ✔ No DNS available
- ✔ Quick hostname mapping
- ✔ Blocking sites (advanced use)
