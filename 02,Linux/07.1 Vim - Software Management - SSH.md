
# #Vim
Vim has modes. This confuses beginners.

**Three Main Modes:**

| Mode         | Purpose                   | How to enter |
| :----------- | :------------------------ | :----------- |
| **Normal**   | Move around, delete, copy | Press `Esc`  |
| **Insert**   | Type text                 | Press `i`    |
| Command mode | save/exit                 | press `:`    |

### Basic Vim Steps
#### Open file:
```
vim filename.txt
```

**To type text:**
- Press **`i`** (now you can type)
- Type your text
- Press Esc when done typing
  
**To save and quit:**
- Press Esc
- Type **`:wq`**
- Press Enter
  
**To quit without saving:**
- Press Esc
- Type **`:q!`**
- Press Enter

#### Vim Cheat Sheet 
| Action                         | Keys              |
| :----------------------------- | :---------------- |
| **Start typing** (Insert mode) | `i`               |
| **Stop typing** (Normal mode)  | `Esc`             |
| **Save**                       | `:w` + `Enter`    |
| **Quit**                       | `:q` + `Enter`    |
| **Save and quit**              | `:wq` + `Enter`   |
| **Quit without saving**        | `:q!` + `Enter`   |
| **Delete line**                | `dd`              |
| **Undo**                       | `u`               |
| **Search**                     | `/word` + `Enter` |

#### Delete / Copy / Paste 
```
dd     # delete line
yy     # copy line
p      # paste
u      # undo
```

#### Search
```
/word
```

#### Save & Exit
```
:wq    # save and quit
:q!    # force quit
```


# Software management 
**installing, updating, removing**, and **verifying** software on a system

- Instead of manually downloading apps like Windows,
		Linux uses **packages + package managers**

### Package
 
 A package is a compressed file that contains:
- Program files
- Libraries
- Config files
- Metadata

## In RHEL

Package format = **.rpm** 
==**RPM**== = Red Hat Package Manager (individual packages)

**Example:**
```
httpd-2.4.57.rpm
```

#### Why Software Management is Important?

| Reason              | Explanation                      |
| ------------------- | -------------------------------- |
| Security            | Updates fix vulnerabilities      |
| Dependency handling | Auto installs required libraries |
| Stability           | Tested packages                  |
| Automation          | Fast installation                |

## Types of Tools in RHEL
#### 1. Low-level tool

**rpm**
- Works directly with .rpm files
- No dependency resolution

#### 2. High-level tool

==**dnf** (modern)==
- Handles dependencies automatically
- Works with repositories


### RPM (Low-level Tool)

- Install .rpm files
- Query package info
- Verify integrity

#### Install package
```bash
sudo rpm -ivh package.rpm
```

#### Remove package
```bash
sudo rpm -e package
```

#### Query installed packages
```bash
rpm -qa
```


#### Files in package
```bash
rpm -ql bash
```

#### ==Verify package==
```bash
rpm -V bash
```

#### Limitation of RPM

It does NOT handle dependencies

**Example:**
- Install nginx → needs libraries
- RPM won’t install them automatically 



____

### DNF (High-level Tool)

**What DNF does:**
- ✔ Install packages
- ✔ Resolve dependencies
- ✔ Connect to repositories
- ✔ Update system

#### Install
```
sudo dnf install nginx
```

#### Remove
```
sudo dnf remove nginx
```

#### Search
```
dnf search httpd
```

#### Update system
```
dnf update 
dnf upgrade 
or
dnf update && dnf upgrade
```

#### List installed
```
dnf list installed
```

#### ==History==
```
dnf history
```

**Undo history**
```
sudo dnf history undo <id>
```

Reverses that specific transaction / **remove the installed tool/file**

**Example:**
```
dnf install vim 
dnf history                     ( id = 22)
sudo dnf history undo 22        ( It will remove vim )         
```

# Repository (Repo)
A repository = collection of packages stored online/offline

**Types:**
- Online repo (internet)
- Local repo (ISO / internal server)

**Example:** 
```bash
cd /etc/yum.repos.d/
```

# Installation Works (Flow)
```
User → dnf install nginx
      ↓
Check repo
      ↓
Download package
      ↓
Resolve dependencies
      ↓
Install via rpm
```

==DNF uses RPM internally==

# ==Dependency Concept== 

Software needs other software

**Example:**
nginx → needs libc, openssl

**DNF:**
✔ Automatically installs

**RPM:**
❌ Fails if missing

# DNF Modules
Used for different versions of software

**Example:**
```bash
dnf module list nodejs
```

#### Enable Version
```
sudo dnf module enable nodejs:18
```

#### Install
```
sudo dnf module install nodejs:18
```


```
1. dnf search vim             (Searches repositories for packages related to vim)
2. dnf install vim            (Check repos → Download vim → Check dependencies → Install) 
                                                     Installs:vim,required                                                                                                 libraries (auto)

3. dnf remove vim             (Removes the package vim)
4. dnf history                (Shows all past transactions)
5. sudo dnf history undo 1     
```
